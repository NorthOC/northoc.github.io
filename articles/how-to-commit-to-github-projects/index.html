How to Commit to GitHub Projects, the Easy Way

Do you even pull request, bro?

Alright, so at some point, you decide to put on some big boy pants and start making pull requests and commiting to open-source projects to make the world a better place. But I guess this is where the first of many god complexes come to die for many young developers.

For some unknown reason, there is a certain amount of gatekeeping in the developer community as to how to commit to projects properly. It is silly, but ask any two different developers "how do I commit to a github project" and you will get two different answers!

Projects are part of the problem

Each open-source project has its own standard which can usually be found in some commits.md file or a specific directory, dedicated to pull request standards, that can be from a couple of pages to dozens of pages in length. Oh, and each project assumes that you already know how to do all the fancy git magic.

The upsides of this is that you limit spaghetti code and the number of noob devs on the contributor list.

The downsides of this is that you get a small number of contributors.

What's the solution?

To fork the project, change the commit standard and claim it your own.

Just kidding. The solution is to be mindful of these awful practices and stay away from projects that have massive gatekeeping.

How to do pull requests

Alright, jokes aside, heres a simple way to do this stuff (the green lines are cli commands):

  1. Fork the repository you want to contribute to

  2. Clone the fork to your pc with git clone https://github.com/your_username/your_repository name

  3. Create a branch for your feature with git checkout -b branch_name

  4. Add the branch upstream so that github has access to it git remote add upstream https://github.com/your_username/your_repository_name

  5. Edit the code

  6. git add .

  7. git commit -m "message of what you've changed"

  8. git push

Once this is done, open your fork in the browser. There should be two buttons above the files saying "Contribute" and "Sync Fork". Click the contribute button and then "Open pull request".

Write what you've changed, submit it and let the project jannies do the rest of the work. Congrats! You now have the ability to make the world a better place.
